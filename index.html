GIT

Git é um sistema de controle de versão distribuído, amplamente utilizado no desenvolvimento de software para rastrear mudanças no código-fonte durante o desenvolvimento. Ele permite que múltiplos desenvolvedores trabalhem em um projeto de forma simultânea, mantendo um histórico detalhado das alterações feitas ao longo do tempo. Criado por Linus Torvalds em 2005, Git é essencial para o gerenciamento eficiente e organizado de projetos de software.

Conceitos Básicos:

Repositório:

Um repositório Git é onde todo o seu código e histórico de alterações são armazenados. Pode ser local (no seu computador) ou remoto (em um servidor como GitHub, GitLab, Bitbucket).

Commit:

Um commit é uma "foto" do seu código em um dado momento. Ele registra mudanças feitas aos arquivos e permite que você volte a esse ponto específico no tempo se necessário.

Branch:

Branches são ramificações do seu código principal. A branch main (ou master) é a principal, mas você pode criar outras branches para desenvolver novas funcionalidades ou corrigir bugs sem afetar o código principal.

Merge:

O merge é o processo de combinar mudanças de diferentes branches. Geralmente, você trabalha em uma branch separada e, quando o trabalho está completo, faz o merge das mudanças na branch main.

Clone:

Clonar um repositório significa criar uma cópia do repositório remoto no seu computador local, permitindo que você trabalhe localmente.

Pull:

O comando pull é usado para atualizar seu repositório local com as mudanças do repositório remoto.

Push:

O comando push é usado para enviar suas mudanças locais para o repositório remoto, compartilhando seu trabalho com outros colaboradores.
